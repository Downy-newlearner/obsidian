- 휘발유를 만들기 위해 원유에서 추출하듯이 Raw data들을 정제해서 
- 쿼리로 접근할 수 있는 데이터들을 Data Warehouse에 넣는다.
    
- Data Lake는 정형/비정형을 가리지 않고 대규모 데이터를 원시 형태 그대로 저장하여, 필요할 때 가공해서 활용할 수 있도록 한다.
    
- Data Lakehouse는 Data Lake의 유연성과 Data Warehouse의 분석 성능을 결합한 개념으로, 원시 데이터와 정제 데이터를 한 곳에서 관리하면서도 SQL 쿼리 같은 분석 기능을 지원한다.
    
- Data Fabric은 다양한 위치(온프레미스, 클라우드, 하이브리드)에 흩어져 있는 데이터들을 통합적으로 연결하고 관리할 수 있는 아키텍처 접근 방식으로, 일종의 데이터 네트워크 역할을 한다.
    
- Digital Transformation(디지털 전환)은 데이터와 디지털 기술을 기반으로 비즈니스 모델, 조직 문화, 업무 프로세스를 혁신해 경쟁력을 확보하는 전략적 변화이다.
    
- 현재는 Raw data들도 클라우드 환경이나 분산 저장소에 직접 쌓아두고, 필요할 때마다 처리·분석하는 방식으로 점점 더 많이 활용되고 있다.

- RDBMS에는 데이터가 Disk에 write 된다.
- 파일은 데이터가 Disk에 write된다.
- 데이터가 쌓이는건 둘이 똑같은데 '어떻게' 쌓이냐가 다르다.

- 최종 목표는 그 회사가 가지고 있는 데이터를 이용해서 경영진의 의사 결정을 돕는 것이다.

- 고객에 제공한 정보는 데이터 웨어하우스에 저장할 것이다.

- 데이터 레이크를 만드는 이유는 돈이 들어서이다.
- 모든 데이터를 오라클 같은 곳에 넣으려면 너무 비싸기 때문에 그냥 파일로 두는 것이다. 이게 데이터레이크
- 그래서 이걸 표준화, 그룹화를 해야하는 이유가 여기서 나온다.
	- 오라클에 넣으면 알아서 해주는데 비싸니까 직접 해야함.

- 데이터 웨어하우스와 데이터 레이크에서 필요한 데이터를 뽑아서 한 곳에 모으는데 IO가 여러번 발생하지만 빅데이터 페브릭은 딱 한 번 발생한다.
