Question: "- 데이터 웨어하우스와 데이터 레이크에서 필요한 데이터를 뽑아서 한 곳에 모으는데 IO가 여러번 발생하지만 빅데이터 페브릭은 딱 한 번 발생한다.왜?"
---
좋은 질문이에요! 핵심은 **데이터를 모으는 방식의 차이** 때문입니다.

---

### **1. 데이터 웨어하우스 & 데이터 레이크**

- 여러 시스템(예: ERP, CRM, 로그 서버 등)에서 필요한 데이터를 뽑아오려면
    
    - 각 소스에서 **추출(Extract)**
        
    - 변환(Transform)
        
    - 적재(Load) → ETL 과정을 거쳐 **새로운 저장소**(DW나 DL)에 넣습니다.
        
    
- 즉, 데이터를 실제로 **복사하고 옮기면서 저장소가 따로 생기기 때문에**
    
    - IO가 소스 시스템에서 한 번,
        
    - 대상 저장소에 다시 한 번 발생 = 여러 번 IO 발생.
        
    

---

### **2. 데이터 패브릭 (Data Fabric)**

- 데이터 패브릭은 데이터를 **물리적으로 다시 모으는 게 아니라**, 흩어진 데이터 소스를 **가상으로 연결**합니다.
    
- 즉, 소스 데이터는 원래 위치(온프레미스 DB, 클라우드 스토리지 등)에 그대로 있고,
    
    - 사용자가 “한 곳에서 보는 것처럼” 접근할 수 있게 **메타데이터·가상화 계층**이 제공됨.
        
    
- 이 때문에 데이터를 **복사/이동하지 않아도 되므로 IO가 최소 1번**(조회 시점)만 발생.
    

---

### **✅ 결론**

- **DW/DL**: 데이터를 **실제로 옮겨서 쌓아두는 방식** → 여러 번 IO 발생.
    
- **Data Fabric**: 데이터를 **가상으로 통합**해서 필요할 때만 접근 → IO가 한 번만 발생.
