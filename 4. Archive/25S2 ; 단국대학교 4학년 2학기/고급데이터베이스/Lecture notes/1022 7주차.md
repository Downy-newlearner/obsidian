## Apache Spark v4
시험에 안나옴. 교수님도 안 써봄.

v4의 핵심은 apache arrow
	기업에는 다양한 언어로 작성된 작업이 있음. 
	상호간에 데이터를 주고받을 때, 단순히는 한 쪽에서 데이터를 파일로 만들어서 건내주면 됨.
	하지만 서로 지향하는 메모리 아키텍처가 다르다. 그래서 위 방법이 쉽지 않다.
	더 효율적인 방법은 "네트워크" -> 이게 Apache Arrow

**Apache Arrow**
_심드_ 가 탑재되어있음(병렬처리 기술)
OS 레벨에서 컨트롤할 수 있다.
Shared memory 라는 영역이 있는데 이 영역은 스택, 힙 메모리에 중복되지 않는다.
그래서 이 영역은 힙 스택에 비해 느리지만, 디스크/메모리보단 빠르다.
C/C++에서 쉽게 접근 가능한 영역.
