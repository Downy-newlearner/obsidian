Question: "- RDBMS에는 데이터가 Disk에 write 된다.- 파일은 데이터가 Disk에 write된다.- 데이터가 쌓이는건 둘이 똑같은데 '어떻게' 쌓이냐가 다르다.어떻게 달라?"
---
RDBMS(관계형 데이터베이스)와 일반 파일 시스템은 **데이터가 디스크에 쌓인다는 점은 같지만, 그 “쌓이는 방식(저장 구조)”이 다릅니다.**

---

### **1. RDBMS (관계형 데이터베이스)**

- **구조적 저장**:
    
    - 데이터는 **테이블(행·열)** 구조에 맞게 저장됨.
        
    - 스키마(컬럼 정의)에 따라 데이터 형식이 엄격히 관리됨.
        
    
- **인덱스 관리**:
    
    - 검색을 빠르게 하기 위해 **B-Tree, Hash Index** 같은 자료구조로 색인을 자동 유지.
        
    
- **트랜잭션 보장**:
    
    - ACID(원자성, 일관성, 고립성, 지속성) 규칙에 따라 저장됨 → 중간에 끊겨도 데이터 무결성 보장.
        
    
- **저장 방식**:
    
    - 내부적으로는 **페이지(Page)** 단위로 디스크에 기록하고, 버퍼 캐시를 통해 효율적으로 I/O 수행.
        
    

---

### **2. 일반 파일 (File System)**

- **비구조적 저장**:
    
    - 파일은 단순히 바이트(byte)의 연속으로 저장됨. (txt, csv, json 등)
        
    - 스키마나 형식은 운영체제가 신경 쓰지 않고, 사용자가 직접 관리.
        
    
- **인덱스 없음**:
    
    - 파일 내 데이터를 검색할 때는 전부 읽어야 함 → RDBMS보다 검색 속도가 느림.
        
    
- **트랜잭션 보장 없음**:
    
    - 단순히 기록(write)하는 수준이라, 도중에 끊기면 파일 손상 가능.
        
    
- **저장 방식**:
    
    - 파일 시스템이 블록(block) 단위로 디스크에 기록.
        
    

---

### **정리**

- **똑같이 디스크에 write** 되지만,
    
    - **DB**는 “테이블 구조 + 인덱스 + 트랜잭션 관리”를 갖춘 **데이터 관리 시스템**이고,
        
    - **파일**은 그냥 “데이터 덩어리”를 디스크에 놓는 것일 뿐.
        
    

  

👉 그래서 **RDBMS는 검색·갱신·보안·동시성 제어에 최적화**,

반면 파일은 **단순 저장·전송·백업**에 더 가깝습니다.
