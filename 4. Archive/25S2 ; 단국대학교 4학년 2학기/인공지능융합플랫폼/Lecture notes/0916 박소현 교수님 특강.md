
## Hadoop
- 병렬 데이터 처리에는 Hadoop이 사용된다
	- 디스크 기반
	- 느림
	- 스트리밍 별도 구성 필요
	- 로그 처리, 일괄 분석에 사용됨

## Spark
- Hadoop보다 빠른 Spark도 있다.
	- 하둡보대 100배 빠르다
	- 머신러닝, 실시간 이벤트 처리가 가능하다.
	- 파이썬에 라이브러리로 있다.
	- 단점은 인메모리라서 고사양 서버가 필요하다.
	- 복잡한 DAG 구조이다.(성능이 저하되는 수가 있다.)
	- Spark Streaming 내장
	- PySpark API 학습이 필요하는 등 진입장벽이 존재한다.

## Apache Kafka
- 실시간 데이터 스트리밍 플랫폼
- 메시지를 디스크에 저장하고 한 번만 전달한다.


| 용어             | 설명                                    |
| -------------- | ------------------------------------- |
| Producer       | 데이터를 생성하여 카프카로 전송하는 주체(예: 웹서버, 센서)    |
| Broker         | 카프카 서버로, 메시지를 받아 저장하고 관리              |
| Topic          | 데이터를 카테고리별로 분류하는 논리적 단위               |
| Partition      | 토픽을 물리적으로 나눈 단위, 병렬 처리 및 확장성을 위해 사용   |
| Consumer       | 토픽에서 메시지를 읽어가는 애플리케이션 또는 서비스          |
| Consumer Group | 여러 Consumer가 그룹을 이루어 토픽을 분담해서 처리하는 단위 |

- 장점
	- 높은 처리량
	- 확장성
	- 내결함성(복제해서 백업)
	- 지속적 보장(유저가 읽을 때까지 유지할 수 있다.)
- 단점
	- 데이터 처리 자체는 하지 않아서 Spark 등과 연동해야한다.
	- 지나치게 짧은 보존기간 설정 시 데이터 유실 위험
