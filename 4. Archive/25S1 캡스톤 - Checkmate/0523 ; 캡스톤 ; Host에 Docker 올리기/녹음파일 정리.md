
컨테이너를 설정하고 실행하려면 docker-compose.yml 파일이 필요하다.
컨테이너의 포트와 host의 포트 매핑이 필요하다.

예를 들어 플라스크가 5000번 포트를 사용하고 있었으면, 컨테이너를 올리기 전에는 그냥 5000번 포트로 사용하면 된다.



## Docker YAML file
- 메모리 관련된 도커 설정을 YAML 파일에서 할 수도 있다.
```
version: "3.8"

services:

  myapp:

    image: myimage:latest

    deploy:

      resources:

        limits:

          memory: 512M

        reservations:

          memory: 256M
```


## 컨테이너의 포트와 host의 포트 매핑
내부 -> 외부 : 5000 포트
외부 -> 내부 : 8080 포트

예를 들어, Flask가 5000번 포트를 사용한다면, 호스트의 8080번 포트를 Flask 컨테이너의 5000번 포트에 매핑하여 외부 접근이 가능하게 할 수 있다.

즉, 호스트:컨테이너 = 8080:5000
