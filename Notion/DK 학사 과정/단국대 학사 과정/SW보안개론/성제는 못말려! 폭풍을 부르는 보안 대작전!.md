# |IT
- Information Technology (IT)
- 집행 기관에 의한 데이터 또는 정보의 자동 수집, 저장, 조작, 관리, 이동, 제어, 표시, 전환, 교환, 전송, 수신에 사용되는 모든 장비 또는 상호 연결된 시스템
- 데이터와 정보의 처리와 분배를 목적으로 네트워크와 컴퓨터 시스템을 사용 혹은 개발을 수반한 기술
# |Information System
- 정보의 수집, 처리, 유지 관리, 사용 공유, 배포 또는 처리를 위해 구성된 정보 자원들의 집합
- 시스템은 하드웨어, 소프트웨어, 정보, 데이터, 애플리케이션, 커뮤니케이션, 사람을 포함한다.
- 공식 정보 시스템은 정보를 얻기 위해 조직 관행에 의해 확립되고 승인된 절차에 의존해 데이터를 수집, 저장, 조작 및 액세스한다.

# |Security(보안)
- Security
    - 안전한 상태이자 위험으로 부터 자유로운 상태
  
- 위험의 종류
    - threat
    - risk
    - vulnerability
  
- 보안의 특성
    
    |   |   |
    |---|---|
    |**Confidentiality(기밀성)**|자료를 의도한 대로만 공개되어야함|
    |**Integrity(무결성)**|시스템 또는 구성요소가 컴퓨터 프로그램 또는 데이터의 부적절한 수정 또는 파괴를 방지|
    |**Availability(가용성)**|시스템 또는 구성 요소가 작동 가능하고 사용이 필요할 때 액세스 할 수 있어야 함|
    |Identification & Authentication  <br>(식별 & 인증)|사용자, 프로세스, 장치에 대해 유효한 ID인지 확인해야함|
    |Accountability  <br>(책임 추적성)|소프트웨어 자산에 영햘을 미치는 행위가 행위에 대한 책임을 위해 사용자에 의해 추적될 수 있다.|
    |Privacy  <br>(사생활)|행위자가 자신의 정보가 어떻게 사용되는지 이해하고 통제할 수 있어야함|
    
    - CIA = Confidentiality, Integrity, Availability
# |Infromation Security
- 기밀성,무결성, 가용성을 제공하기 위해, 무단 접근, 사용, 공개, 중단, 수정 또는 파괴로부터 정보 및 정보 시스템을 보호하는 것
# |Information Assurance
- 정보 및 정보 시스템의 가용성, 무결성 및 기밀성을 보장하여 정보 및 정보 시스템을 보호하고 방어하는 조치
# |Computer Security
- 컴퓨터에서 처리 및 저장되는 정보의 기밀성, 무결성 및 가용성을 보장하는 조치/대책 및 제어/통제
# |Cybersecurity
- 가용성, 무결성, 인증, 기밀성 및 부인 방지를 보장하기 위해 컴퓨터, 전자 통신 서비스, 유선 통신 및 전자 통신의 손상 방지, 보호 및 복원
- 정보 및 통신 시스템, 그 안에 포함된 정보가 손상, 무단 사용이나 변조 또는 악용되는 것으로부터 보호 및/또는 방어되는 활동 또는 프로세스, 능력, 또는 상태.

    - 시스템에 결함이 발생하기 쉽다.
    - 공격자가 결함에 접근할 수 있다.
    - 공격자가 결함을 이용할 수 있다.
- 기밀성, 무결성 또는 가용성의 실패로 이어질 수 있는 응용 프로그램, 시스템, 장치 또는 서비스의 버그, 결함, 취약성 또는 노출

# |Log4Shell
- 악의적인 공격자가 서버를 제어하는 임의의 java 코드를 실행할 수 있게하는 원격코드 실행 취약점

- Log4Shell은 다음과 같은 다른 위협으로 이어질 수 있음
    - 코인마이닝
        - 공격자가 피해자의 리소스를 활용하여 암호화폐를 채굴할 수 있음
    - 네트워크 서비스 거부(DoS)
        - 공격자가 피해자의 네크워크 서비스를 종료 및 비활성화하여 접속할 수 없게 함
    - 랜섬웨어
        - 공격자가 몸값을 목적으로 데이터를 수집하고 암호화

# |Online Bank에서의 보안
- Online Bank 운영자의 보안 문제
    - 모든 계정에 대한 개인정보 보호 및 데이터 보안
  
- 보안 타입에 따른 위협
    - 기밀성
        - 패킷 캡쳐, 키로거등으로 ID/PW 갈취
    - 무결성
        - 계좌정보를 바꿈
        - 개인 계정정보를 변경
    - 가용성
        - DDOS
        - 온라인 거래와 관련된 보안
  
- 사용자와 운영자 사이의 보안 걱정 비교
    
    |공통|계정 보안|
    |---|---|
    |운영자|데이터 기밀성 (고객, 거래 정보)  <br>시스템 무결성 (무단변경 조작X)|
    |사용자|계정보안(자격증명, 개인정보보안)  <br>거래보안  <br>가용성의 침해|
    
- 공격자의 시선
    - 취약점을 이용해 자산, 계좌번호, PIN번호등 개인정보를 탈취
- Threat : 공격자가 언제든지 Attack할 수 있는 가능성

# |Phising
- 개인정보(Private data) + 낚시(Fishing)
- 이메일이나 웹 사이트에서 사기성 권유를 통해 은행 계좌 번호, 신용카드 번호, 패스워드와 같은 민감한 데이터를 획득하려고 시도하는 수법으로, 가해자는 합법적인 기업이나 평판이 좋은 사람으로 가장함
- 피싱은 사회공학적 기법이다.
# |Social Engineering (사회 공학)
- 개인을 속여 민감한 정보를 누설하게 하거나, 무단 접근 권한을 얻거나, 확신과 신뢰를 얻기 위해 개인과 교제하여 사기를 치는 행위
- 대상에 대한 심리학적 접근을 바탕으로 한다.

# |Man-in-the-middle attack

# |DoS and DDoS
- 서버에 오버플로 액세스 요청을 전송하여 특정 시스템/웹 사이트에 액세스하기 위해 전송되는 검증된 리소스를 멈추는 것이다.
- 서비스를 사용하지 못 하게 하는 것이다.
# |Pharming
- 기술적 수단을 사용하여, 합법적인 사이트로 가장하여 개인 정보를 누설하는 가짜 웹 사이트에 액세스하도록 사용자를 리디렉션하는 것
- 공격자가 DNS와 같은 인프라 서비스를 손상시켜서, 가입자를 위조된 검증자/RP로 잘못 안내하여 가입자가 민감한 정보를 공개하거나, 유해한 소프트웨어를 다운로드 하거나, 또는 사기 행위에 말려들게 할 수 있는 공격
# |Cross Site Scripting (XSS)
- 양성 웹 사이트에 공격자가 악성 코드를 주입할 수 있는 취약점.
- 웹 사이트와 클라이언트 간의 데이터 전송의 기밀성과 무결성을 손상시킬 수 있다.
- 권한이 없는 사용자가 클라이언트 측 스크립트를 주입하여, 웹 어플리케이션의 다른 사용자에 의해 그 스크립트가 실행될 수 있게 하는 웹 어플리케이션에서 발견된 보안 결함
# |Drive-by Downloads

# |Advanced Persistance Threat (APT) : 지능형 지속 위협
- 사이버, 물리적 및 속임수를 비롯한 여러 공격 벡터를 사용하여, 공격 목표를 달성할 기회를 만들 수 있는 정교한 수준의 전문 지식과 중요한 자원을 보유한 적
- 공격 목표에는 정보를 유출하거나 임무, 프로그램 또는 조직의 중요한 측면을 약화시키거나 방해하기, 또는 미래에 공격 목표를 수행하기 위해 공격자 자신의 위치를 설정할 목적으로, 대상 조직의 IT 인프라 내에 발판을 마련하고 확장하는 것이 포함됨
- APT는 장기간에 걸쳐 공격 목표를 반복적으로 추구
- 공격에 저항하는 방어자의 노력에 적응하여, 공격 목표를 실행하는 데 필요한 상호 작용 수준을 유지
# |Zero-day Attack
- 이전에 알려지지 않은 하드웨어, 펌웨어 또는 소프트웨어 취약성을 이용하는 공격
# |Cryptojacking
- 다른 사람의 컴퓨터를 장악해서 암호화폐를 채굴하는 행위
# |Tunneling
- 한 네트워크가 다른 네트워크의 연결을 통해 데이터를 전송할 수 있도록 하는 기술
# |DNS Tunneling
- DNS 프로토콜을 이용하여 클라이언트-서버 모델을 통해 멀웨어 및 기타 데이터를 터널링 한다.
# |RaaS 조직
- **Ransomware as Service**
- 랜섬웨어를 서비스 하는 조직
# |Supply Chain Attack
- 연쇄적인 공급망 공격

- ex) spoofing, replay attack
- 사용자, 자격 증명 위조, 이메일 스푸핑, 가짜 메시지, 이전에 가로챈 메시지를 다시 보냄(replay attack), DNS spoofing, Pharming, Phising, Telephone scam

|Tampering (변조)|응용 프로그램과 합법적인 사용자 간에 교환된 데이터를 수정|매개 변수 또는 값을 변경에 의해 메시지 무결성이 손상됨|

|Elevation of Privilege (권한 상승)|공격자는 권한 없는 수단을 통해 높은 액세스 권한을 얻는 것|- 루팅  <br>- 공격자가 그룹원 자격을 변경|
# |Repudiation (부인)
- 사용자가 액션 수행을 거부하지만 액션의 대상이 그렇지 않음을 증명할 방법이 없을 때 발생
![[Source/Untitled 20 2.png|Untitled 20 2.png]]
- 사용자의 책무성(접근, 위치, 관리, 보안)에 대한 반박
# |Elevation of Privilege (권한 상승)
![[Source/Untitled 21 2.png|Untitled 21 2.png]]
- 사용자가 의도하거나 사용할 수 있는 권한 이상의 권한 또는 권한에 대한 불법적인 액세스를 획득하는 것을 포함하는 공격
- set-uid 루트 프로그램에 대한 버퍼 오버플로 취약성
    - 관리자 권한으로 실행된 파일에서 버퍼 오버플로우를 발생시켜 관리자 권한을 유지시키는 취약점
    - sudo chown gdhong program.c
        - 파일 소유자를 변경
    - su : 계정을 관리자로 변경
    - sudo : 관리자 권한으로 실행
  
# |Asset, Asset identification
- Asset
    - 우리가 지키고 싶은 것, 조직에 가치가 있는 모든 것
    - H/W, S/W, Information, Data, Person,…
  
- Asset identification
    - 주어진 자산을 고유하게 식별하는 데 필요한 속성 및 방법.
# |Threat
- 정보의 무단 접근, 파기, 공개, 수정 및/또는 서비스 거부를 통하여, 조직 운영(임무, 기능, 이미지 또는 평판 포함), 조직 자산 또는 개인에 부정적인 영향을 미칠 가능성이 있는 모든 상황 또는 사건
- 위협 소스가 특정 정보 시스템 취약점을 성공적으로 악용할 **가능성**
# |Attack
- 위협을 이행/완수하기 위해 취약점을 악용하는 행동/행위/조치
- 컴퓨터 자원의 기밀성, 무결성, 책임추적성, 또는 가용성에 영향을 미치는 특정 **위협의 실현**
# |Control, Security Control
- Control
    
    - 취약성 완화 또는 제거
    - 행정적, 기술적, 관리 또는 법적 특성을 가질 수 있는 정책, 절차, 지침, 관행 또는 조직 구조를 포함하여, 위험을 관리하는 수단.
    
      
    
- Security Control
    - 정보의 기밀성, 무결성 및 가용성을 보호하고 일련의 정의된 보안 요구 사항을 충족하도록 설계된, 정보 시스템 또는 조직에 대해 규정된 안전장치 또는 대책
# |Weakness
- 특정 상황에서 취약점 도입에 기여할 수 있는 S/W, 펌웨어, H/W 또는 서비스 구성 요소의 상태
- ex ) 스팸을 거르지 않는 공개 이메일 서버
# |CWE common Weakness Enumeration
- Vulnerability가 될 수 있는 SW 및 HW Weakness 목록
# |CVE common Vulnerability Enumeration
- Vulnerability가 정의되어 있음
- Vulnerability는 다음과 같은 방법으로 경감될 수 있음
    - 코드 변경을 수반
    - 사양 변경이나 사양 저하도 가능하다.
    - 영향을 받는 프로토콜이나 기능 전체의 제거
- CVE-ID : CVE 프로그램에 의해 할당된 고유한 영숫자 식별자
    - CVE-연도-발행번호순
- NVD (National Vulnerability Database)
    - 미정부에서 관리하는 vulnerability 데이터 베이스
    - CVE보다 더 구체적이다.

# |Security Threat
- 회사/개인의 자산에 대한 공개, 변경, 손실, 손상 또는 가용성을 야기할 수 있는 모든 행동
  
- 세가지 주요 구성요소
    - Target : 공격당할 수 있는 조직의 자산
    - Agent : 의도적이거나 비의도적인 위협에서 비롯된 사람 혹은 조직 (없을 수 도 있다.)
    - Event : 대상의 취약성을 이용하는 행동
- 이벤트가 없으면 위협도 없다!
# |Example of Threats
- Threat without Agent
    - Asset : 백업 되지 않은 서버
    - Event : 서버 룸에 홍수나 화재
- 외부 vs 내부, 고의 vs 우연
    - Asset
    - Agent : 내부 혹은 외부
        - ex) 새로운 소프트를 설치하러온 내부 사용자가 비밀번호 바꾸는 걸 까먹음
    - Event : 고의 혹은 우연
- Attack
    - Asset
    - Agent : 고의적으로 위협 이벤트 실행
    - Event : 고의
