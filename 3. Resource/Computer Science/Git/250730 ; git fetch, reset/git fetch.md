## git fetch

## : 원격 저장소의 변경 사항을 

## 로컬로 가져오기

  

### **📌 개념**

- 원격 저장소의 변경 사항을 **로컬 저장소로 가져오기만 하고**, **현재 작업 브랜치는 그대로 유지**됩니다.
    
- 실제 코드에 영향을 주지 않고, 원격 상태를 최신으로 동기화할 때 사용합니다.
    

  

### **🧪 예시**

```
git fetch origin
```

- origin에서 최신 변경 사항을 받아오지만, HEAD나 작업 디렉토리는 바뀌지 않음.
    

  

### **🔍 사용 목적**

- git pull보다 안전한 방식으로 원격 변경 사항을 확인하고 싶을 때
    
- 변경 내용을 직접 확인하고 수동으로 [[git merge|merge]]/[[git rebase|rebase]] 하고 싶을 때
    

---

## **📌 요약 비교표**

| **명령어**   | **주요 목적**     | **현재 브랜치 이동** | **파일 변경** |
| --------- | ------------- | ------------- | --------- |
| git fetch | 원격 변경 사항 확인용  | ❌             | ❌         |
| git reset | 커밋/브랜치/상태 초기화 | ✔             | 옵션에 따라 다름 |
