### 문제 상황
- GitHub에 SSH를 통해 연결할 때 "Permission denied (publickey)" 오류 발생.
- GitHub 서버의 신원을 확인할 수 없다는 경고 메시지 발생.
- 일단 무엇보다 자꾸 Username이랑 Password 입력해야하는게 너무 귀찮고 내가 모르는 깔끔한 방법이 있을 것이라고 생각했다.

### 해결 방법

1. **SSH 키 생성 및 설정**:
   - SSH 키 쌍 생성: `ssh-keygen -t rsa -b 4096 -C "your_email@example.com"`
   - SSH 에이전트 시작: `eval "$(ssh-agent -s)"`
   - SSH 키 추가: `ssh-add ~/.ssh/id_rsa`

- ssh-keygen과 ssh-add는 [[OpenSSH]]에서 제공하는 명령어이다.

2. **GitHub에 SSH 공개 키 등록**:
   - 공개 키 확인: `cat ~/.ssh/id_rsa.pub`
   - GitHub 설정에서 `SSH and GPG keys`로 이동하여 `New SSH key` 추가.
   - 공개 키를 붙여넣고 저장.

3. **SSH 연결 설정**:
   - Git 리모트 URL을 SSH로 변경: `git remote set-url origin git@github.com:username/repository.git`

4. **서버 신원 확인**:
   - 처음 연결 시 서버의 공개 키 지문을 확인하고 "yes" 입력하여 신원을 수동으로 확인.

### 추가 팁
- 개인 키(`id_rsa`)는 안전하게 보관.
- 공개 키(`id_rsa.pub`)는 서버에 등록.
- SSH 키를 사용하면 비밀번호 없이 안전하게 인증 가능.
